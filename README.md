Мобильная конфигурация, сделана в дополнение к основной конфигурации. 
В мобильной конфигурации реализованы интеграционные механизмы (REST API) 

Механизм аутентификации. 
Отправляет в основную конфигурацию логин в формате JSON.
Обращение к Web сервису user с помощью GET запроса. 
Основная конфигурация проверяет совпадение логина с реквизитом
"ПользователиМП" справочника Контрагенты, и в формате JSON
возвращает значение типа булево. После положительного ответа, 
создаётся пользователь и открывается форма, которая доступна для роли
авторизованный пользователь. 
( для теста в поле логин нужно ввести "Админ") 

Механизм отправки данных формы для создания документа записи клиента
После аутентификации, и заполнения данных формы записи, 
по нажатию кнопки записать, формируется JSON и 
POST запросом передаётся с мобильной конфигурации в
Основную конфигурацию.
Обращение к Web сервису customerOrders. 
Основная конфигурация создаёт и проводит
документ записи клиента, возвращает ответ 200, в случае
возврата 200 мобильная конфигурация тоже создаёт и проводит документ
и выдает сообщение об успешной записи клиента.
